Cómo registran tus trabajadores

StarX usa un único bot compartido, ya configurado por la plataforma. Tú solo lo enciendes o apagas aquí — no hay tokens que gestionar.

  1. Carga a tus trabajadores con su teléfono (en Trabajadores o por importación/API).
  2. El trabajador abre el bot y pulsa /start.
  3. Comparte su número de teléfono y queda enlazado automáticamente.
  4. Pulsa /route, elige una parada y hace su check-in con ubicación y foto.

Si el botón Encender bot aparece deshabilitado, el operador de la plataforma aún no ha configurado el token del bot en el servidor.

Conexión por API (conector)

El cliente empuja su catálogo y lee las visitas por API. Se autentica con la cabecera X-API-Key. Estado: …

Esta es la clave única de tu empresa. Entrégasela a quien integre tu sistema. Si la regeneras, la anterior deja de funcionar de inmediato.

{
  "workers": [
    { "name": "Ivan Petrenko", "phone": "+380671112233" },
    { "name": "Olena Kovalenko", "phone": "+380677654321" }
  ]
}
{
  "points": [
    { "id": "P1", "name": "Silpo Khreshchatyk", "address": "Khreshchatyk St 15, Kyiv", "workerPhone": "+380671112233" },
    { "name": "ATB Podil", "address": "Verkhnii Val St 24, Kyiv", "workerPhone": "+380677654321" }
  ]
}

workerPhone (opcional) enlaza el punto con su trabajador asignado. Las coordenadas se rellenan solas en el primer check-in de cada punto.

curl -H "X-API-Key: TU_CLAVE" \
  https://TU-DOMINIO/api/v1/visits?limit=500

Preparar un Google Sheet

Crea las pestañas workers, points y visits (con cabeceras) en tu hoja conectada. Se puede ejecutar de nuevo sin riesgo — nunca borra datos.

Estructura del Excel/CSV

Puedes usar cualquier cabecera — al importar asignas cada columna. Ejemplos recomendados:

id,name,address,workerPhone
P1,Silpo Khreshchatyk,Khreshchatyk St 15,+380671112233
,ATB Podil,Verkhnii Val St 24,+380677654321

La columna del teléfono del trabajador es opcional: enlaza cada punto con su trabajador asignado.

name,phone
Ivan Petrenko,+380671112233
Olena Kovalenko,+380677654321
